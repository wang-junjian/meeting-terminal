# 智会 T1 (SmartMeet T1) · 硬件说明书

> AI 智能会议系统桌面终端 — Rev A（3D 打印打样）
> 适用场景：领导会议室（≤20 人），实时语音转写 / 说话人分离 / 会议纪要，全离线部署
> 计算平台：NVIDIA Jetson AGX Thor 开发套件（T5000，128 GB）

---

## 1. 概述

智会 T1 是「AI 智能会议系统」的配套桌面硬件终端，放置于会议桌中央，完成
6+1 麦克风阵列拾音、扬声器播报、圆形状态屏显示、4K 视频采集（带物理隐私滑盖），
全部 AI 推理在机内 Jetson AGX Thor 上离线完成，无外网依赖。

整机为跑道形（stadium）水平截面的立式桌面终端，顶盖 12° 前倾，
造型语言为沉稳政企风。Rev A 采用 3D 打印打样，壁厚 2.4 mm，
零件间用 M3 内六角螺丝 + 热熔铜螺母装配。

## 7. 装配说明

1. **热熔铜螺母预埋**：底壳顶部 4 个螺丝柱（Φ4.2 孔）、内框/饰圈对应孔位，
   烙铁 230 °C 压入 M3 铜螺母。
2. **内框装入底壳**：内框支架（02）从底壳开口放入，落位于 z=20 承台，4 点 M3 固定。
3. **开发套件安装**：4 根 M3×10 铜柱拧入内框铜柱孔（默认 96×96，可调），
   Thor 套件落位后 M3 螺丝固定；接好风扇与 Micro-Fit 电源线。
4. **USB 音频小板**：固定于内框边缘铜柱，USB 线接 Thor。
5. **拾音仓电子件预装到顶盖（03）**：麦克风阵列板对准 7 个 Φ2.5 孔贴合；
   圆形屏模组沉入 Φ72 视窗；LED 灯环板对准 12 个导光孔；
   扬声器装到顶盖内两支架块之间（轴线法向斜面、朝前）；
   摄像头模组对准 Φ8 孔，滑盖（05）卡入滑槽。
6. **合盖**：顶盖对准底壳上沿，4 × M3 螺丝穿入四角铜螺母锁紧。
7. **屏幕装饰圈（04）**：压入屏幕视窗与摄像头孔外缘（点胶或 3M 胶）。
8. **脚垫**：4 × Φ20 橡胶脚垫贴入底部凹槽。
9. **整机检查**：通电前确认无螺丝松动、线材不压风扇；首次开机进 JetPack 烧录镜像。

## 8. 散热与声学设计说明

**散热**：Thor 套件 40–130 W，自带主动风扇。整机风道为「下进、后上出」：
左右侧壁下部各 6 条 3×20 进风百叶（z=12–50），冷空气进入主机仓下部，
经开发套件散热器后，热空气由背部上沿 3×7 Φ4 排风孔阵列（z=88–104，
正对套件风扇上沿高度）排出。拾音仓电子件功耗 <10 W，依靠壳体自然对流。
Rev B 建议在排风孔内侧加防尘网，并按实测温升决定是否加 40 mm 辅助风扇。

**声学**：
- 麦克风环 Φ70 布置在 12° 斜面上，拾音轴朝向前上方与会者头部平齐（桌面 750 mm + 斜面倾角补偿坐姿声源入射角），6 麦均布支持 360° 波束成形，圆心参考麦用于通道校准。
- 扬声器与麦克风分仓：扬声器在拾音仓前部朝前发声，与麦克风孔保持 ≥90 mm 间距，配合 XVF3800 的 AEC 抑制回采。
- 出声孔 36 × Φ2 阵列开孔率约 18%，兼顾透声与外观。
- 整机重心低（Thor 套件在底部 1/3 区域），跑道形底面 + 4 脚垫，避免桌面振动传导至麦克风。

## 9. 设计决策备注

规格未覆盖或执行中需要取舍的点，记录如下：

- **D1 · Thor 套件占位尺寸（重要）**：经核实，NVIDIA Jetson AGX Thor 开发套件
  官方整机为 **243.19×112.40×56.88 mm**（含脚垫、载板、模组与散热），功耗 40–130 W，
  载板电源输入 Micro-Fit 9–28 V，官方随附 140 W 适配器。其 243 mm 宽度
  **无法放入 190 mm 宽壳体**。Rev A 按任务书以 110×110×65 mm 占位块完成结构 /
  散热 / 声学 / 文档全链路验证；**Rev B 建议二选一**：(a) 改用 T5000 模组
  （100×87 mm）+ 自研载板，可维持现有外形；(b) 壳体加宽至约 260 mm 以容纳官方套件整机。
- **D2 · LED 灯环位置**：规格「顶盖前缘一圈 LED 灯环」。因 12° 斜面上屏幕（Φ87 模组）、
  麦克风环（Φ70）与出声孔阵列的空间竞争，12 个 Φ3 导光孔最终以 R45 环绕屏幕布置，
  形成「状态光环」，视觉效果与可读性更好，灯效语义不变。
- **D3 · 出声孔位置**：顶盖前缘垂直带仅约 31–46 mm 高，放不下 Φ52 扬声器正面。
  扬声器法向斜面、朝前上方安装于拾音仓前部，出声孔阵列开在斜面前缘（扬声器正前方），
  声学上朝向与会者，与「朝前安装」意图一致。
- **D4 · 斜面布局**：屏幕视窗居中偏左（-40, 18）、麦克风环在右后（42, -8）、
  摄像头在右前（42, 42）、隐私滑盖在摄像头上方。该布局是在 150 mm 进深内同时容纳
  Φ87 屏模组 + Φ70 麦环 + 出声孔的唯一无干涉解（逐项碰撞已验证）。
- **D5 · 弧形背板接口**：跑道形壳体背部直边仅 40 mm，放不下全部接口。
  利用 R75 大弧面（16 mm 宽端口处弦高仅 0.4 mm）将 9 个接口沿背部弧线展开布置，
  保持跑道形外观不变，每个端口法向出壳，插拔无遮挡。
- **D6 · FreeCAD 参数化机制**：FreeCAD 1.1 无界面模式下 PartDesign Body 挂接脚本实体
  不稳定（实测 Body 状态 Invalid），故每个零件采用 **App::Part 容器 + Part 布尔实体**。
  全部尺寸集中于 `Master` 电子表格（别名单元格），几何由 `build.py` 从表格取值生成；
  改型流程：在 FCStd 中编辑 Master → 保存 → `freecadcmd build.py from-fcstd` 全量重建并重导出。
  文档内 `README_Parametrics` 对象亦记录了该流程。
- **D7 · 配色渲染映射**：渲染图中顶盖近黑件按 #2E3034 提亮（#1A1B1D 在米白底下
  会丢失全部细节），实物仍以 #1A1B1D 为准。

## 10. 复现指南

```bash
cd meeting_terminal
# 从零建模 + 导出 + 校验（约 1–2 分钟）
/Applications/FreeCAD.app/Contents/Resources/bin/freecadcmd build.py
# 仅校验已保存的 FCStd（重开 + recompute + 实体有效性）
/Applications/FreeCAD.app/Contents/Resources/bin/freecadcmd build.py verify-only
# 按 FCStd 中 Master 表格参数改型重建
/Applications/FreeCAD.app/Contents/Resources/bin/freecadcmd build.py from-fcstd
# 重新生成 5 张渲染图
python3 make_renders.py all        # 或单张: iso_front / dimensions / back_interface / exploded / internal_layout
```

文件清单：

| 文件 | 说明 |
|------|------|
| `SmartMeet_T1.FCStd` | 参数化装配模型（Master 表格 + 6 个零件容器 + 7 个内部占位组件） |
| `build.py` | FreeCAD 建模/导出/校验脚本（可重入） |
| `render_lib.py` / `make_renders.py` | 渲染管线（纯 Python STL 解析 + matplotlib） |
| `export/SmartMeet_T1_Assembly.step` | 整机装配 STEP（5 个打印件） |
| `export/SmartMeet_T1_Full.step` | 含内部占位组件完整 STEP |
| `export/parts/*.step / *.stl` | 各零件独立 STEP + STL（0.04/0.06 网格精度，可直接切片打印） |
| `export/anchors.json` | 渲染标注锚点（端口/零件坐标，由 build.py 生成） |
| `renders/*.png` | 5 张文档渲染图（米白底 #F5F1E8，≥1600 px） |
| `BOM.csv` / `BOM.md` | 物料清单 |
